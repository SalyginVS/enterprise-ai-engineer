## Файл 01_raw_http.py

### 🎯 Суть лабораторной работы (Файл 01)
Цель этого скрипта — показать, как выглядит **самый базовый, "сырой" запрос** к большой языковой модели (LLM). Здесь не используются специальные библиотеки для работы с ИИ (как OpenAI или LangChain). ### 🐍 Разбор синтаксиса Python по шагам

```python
import requests
import json
import time
```
* **`import`** — команда для подключения внешних библиотек (модулей) в наш код.
* `requests` — популярная библиотека для отправки запросов по сети (в интернет или локальную сеть).
* `json` — библиотека для работы с форматом данных JSON (стандартный формат для передачи данных между серверами).
* `time` — встроенный модуль для работы со временем.

```python
OLLAMA_HOST = "http://192.168.0.128:11434"
```
* Создается переменная (константа) `OLLAMA_HOST`, содержащая строку (`string`) с адресом сервера, где запущена нейросеть (Ollama). Имена констант принято писать ЗАГЛАВНЫМИ буквами.

```python
def first_llm_call():
    """Первый вызов к LLM через сырой HTTP запрос"""
```
* **`def имя_функции():`** — так в Python создаются функции (блок кода, который можно вызывать по имени).
* Текст в тройных кавычках `"""..."""` сразу под названием функции — это **docstring** (строка документации). Она не исполняется, а служит подсказкой-описанием того, что эта функция делает.

```python
    print("Отправляю запрос к модели qwen3:8b...")
    start_time = time.time()
```
* `print()` выводит текст на экран (в консоль).
* `time.time()` возвращает текущее время (в секундах). Мы сохраняем его в переменную `start_time`, чтобы потом посчитать, сколько времени модель генерировала ответ.

```python
    payload = {
        "model": "qwen3:8b",
        "messages": [
            {
                "role": "user",
```

* `payload` — это переменная типа **словарь** (`dict` или dictionary). Словари в Python создаются фигурными скобками `{}` и хранят данные в виде пар "ключ: значение".
* Внутри ключа `messages` лежит **список** (`list`, в квадратных скобках `[]`), внутри которого находится еще один словарь. Это стандартный формат общения с чат-ИИ: мы передаем историю сообщений, где у каждого сообщения есть `role` (кто говорит) и `content` (что говорит).
* `#` означает комментарий — этот текст игнорируется при выполнении программы.

* Вызываем функцию `post` из библиотеки `requests`, чтобы отправить данные на сервер. 
* Буква `f` перед строкой (`f"{OLLAMA_HOST}..."`) делает строку **форматированной (f-string)**. Это позволяет вставлять значения переменных прямо внутрь текста с помощью фигурных скобок `{}`.
* Аргумент `json=payload` говорит библиотеке `requests` автоматически превратить наш словарь в JSON-формат перед отправкой.
* То, что ответит сервер, сохраняется в переменную `response`.

```python
    elapsed = time.time() - start_time
```
* Снова берем текущее время и вычитаем время начала (`start_time`), получая время, затраченное на выполнение запроса (в переменную `elapsed`).

```python
    if response.status_code == 200:
```
* **`if`** — условный оператор ("если"). Код внутри сработает только если код ошибки HTTP (статус) равен `200` (что в интернете означает "ОК / Успешно"). Оператор `==` проверяет равенство.

```python
        data = response.json()
        answer = data["message"]["content"]
```
* Если все прошло успешно, мы вызываем метод `.json()` у ответа сервера, который превращает пришедший от сервера текст обратно в словарь Python (`data`).
* Затем мы "ныряем" в этот словарь, чтобы достать сам текст ответа: берем ключ `"message"`, а внутри него ключ `"content"`. Сохраняем это в `answer`.

```python
        print(f"\n--- Ответ модели ---")

        print(f"\n--- Статистика ---")
```

* В f-строке `{elapsed:.1f}` означает: возьми число `elapsed` и отформатируй его как дробное число (float, буква `f`), оставив только **1 знак после запятой** (`.1`).

```python
    else:
        print(f"Ошибка: {response.status_code}")
        print(response.text)
```
* **`else`** ("иначе") — блок кода, который выполнится, если условие `if` оказалось ложным (то есть сервер вернул ошибку, статус не 200). Выводит код и текст ошибки, чтобы мы могли понять, что пошло не так.

```python
if __name__ == "__main__":
    first_llm_call()
```
* Это стандартная для Python конструкция ("точка входа" в программу). Она означает: "Если этот файл был запущен напрямую (а не импортирован как модуль в другой файл), то нужно выполнить вызов функции `first_llm_call()`".

## Файл 02_ollama_library.py

### 🎯 Суть лабораторной работы (Файл 02)
В предыдущем файле мы отправляли ИИ "сырые" HTTP-запросы вручную. Это полезно для понимания "под капота", но писать такой код долго и неудобно. 

Цель этой лабораторной — показать, как сделать **то же самое, но с использованием специальной библиотеки-обертки `ollama`**. Библиотека берет всю черновую работу (формирование JSON, обработку HTTP-статусов, разбор ответа) на себя. Код становится короче, чище и понятнее!

---

### 🐍 Разбор синтаксиса Python по шагам

```python
from ollama import Client
import time
```
* **`from ... import ...`** — это еще один способ подключения библиотек. Вместо того чтобы импортировать всю библиотеку `ollama` целиком, мы загружаем **только один конкретный класс (инструмент)** под названием `Client`.
* `time` подключаем как и раньше, чтобы замерять скорость ответа (секундомер).

```python
# Подключаемся к серверу лаборатории
client = Client(host="http://192.168.0.128:11434")
```
* Здесь мы создаем **объект** (или экземпляр) класса `Client`. Это как купить телефон: `Client` — это чертеж телефона, а `client` (с маленькой буквы) — это наш конкретный рабочий аппарат.
* В скобках мы передаем аргумент `host="http://..."`. Этим мы сообщаем нашему "клиенту", по какому сетевому адресу искать языковую модель.

```python
def call_with_library():
    """Вызов через официальную библиотеку ollama"""
    
    print("Отправляю запрос через ollama library...")
    start_time = time.time()
```
* Снова объявляем функцию, пишем документацию (`docstring`) и запускаем наш секундомер с помощью `time.time()`.

```python
    response = client.chat(
        model="qwen3:8b",
        messages=[
            {
                "role": "user",
                "content": "Объясни что такое AI Agent в 2-3 предложениях. Отвечай по-русски."
            }
        ]
    )
```
* **Это самое главное отличие!** Вместо того чтобы возиться со словарями `payload` и библиотекой `requests.post()`, мы просто вызываем метод `.chat()` у нашего объекта `client`.
* В этот метод мы передаем название модели (`model`) и историю переписки (`messages`).
* Под капотом метод `client.chat()` сам соберет JSON, сам отправит HTTP-запрос (как в 1-м файле) и сам обработает ответ. Мы об этом даже не думаем!
* Результат сохраняется в переменную `response`.

```python
    elapsed = time.time() - start_time
```
* Останавливаем секундомер и считаем потраченное время.

```python
    # Доступ к ответу через объект
    print(f"\n--- Ответ модели ---")
    print(response.message.content)
```
* Еще одно отличие: библиотека `ollama` возвращает нам не просто большой словарь (как это было с `.json()` в предыдущем файле), а **удобный объект**. 
* Вместо неуклюжего доступа по ключам через словари `data["message"]["content"]`, мы используем **точечную нотацию (dot-notation)**: `response.message.content`. Это читается гораздо легче: "Взять у ответа (`response`) его сообщение (`message`) и прочитать текст (`content`)".

```python
    print(f"\n--- Статистика ---")
    print(f"Время ответа: {elapsed:.1f} сек")
```
* Форматированный вывод (f-строка) с округлением до одного знака после запятой (`.1f`).

```python
if __name__ == "__main__":
    call_with_library()
```
* Проверка "точка входа": если запускаем этот файл напрямую, вызываем нашу функцию `call_with_library()`.

## Файл 03_token_stats.py

### 🎯 Суть лабораторной работы (Файл 03)
В этой лабораторной мы учимся **заглядывать под капот ответов нейросети и считать токены**. Токен — это базовый "кусочек" текста, с которым работает ИИ (часть слова, целое слово или знак препинания). 

Этот скрипт важен для двух вещей:
1. **FinOps (финансовый контроль):** API коммерческих моделей берут деньги за каждый обработанный токен. Уметь их считать = уметь предсказывать затраты.

## Файл 06_chain_of_thought.py

### 🎯 Суть лабораторной работы (Файл 06)
Здесь мы изучаем технику **Chain-of-Thought (CoT) — Цепочка рассуждений**.
Если задача сложная, обычный запрос возвращает плохой результат, так как мы требуем от нейросети сразу выдать готовый ответ-JSON, не дав ей "подумать". Техника CoT заставляет модель сначала вслух перечислить признаки, проанализировать их, и только затем выдать финальное решение.

---

### 🐍 Разбор синтаксиса Python по шагам

```python
SYSTEM_COT = """Ты — эксперт по классификации...
Рассуждай пошагово:
<thinking>
...
</thinking>
<answer>
...
</answer>"""
```
* Используем XML-подобные теги, чтобы жестко разделить процесс: в `thinking` пойдет текст с рассуждениями модели, а в `answer` - строго JSON с финальным решением.

```python
    if "<thinking>" in raw and "</thinking>" in raw:
        thinking = raw.split("<thinking>")[1].split("</thinking>")[0].strip()
```
